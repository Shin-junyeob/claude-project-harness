🏗️ new-project.sh — 하네스 가이드

전역 디렉토리는 새 프로젝트를 찍어내는 금형(템플릿)입니다. 공통 규칙·훅·도구·자율 실행 루프를 미리 갖춰두고, 한 번의 명령으로 새 프로젝트 폴더에 복제합니다.

$ ./new-project.sh <프로젝트명>

① 한눈에

무엇을 하나

전역의 공통 세팅을 새 폴더로 복제 → git(main/stable) 초기화 → pre-commit 활성화 → (gh 있으면) GitHub private 리포 생성·푸시.

무엇을 위해

모든 프로젝트가 동일한 규칙·안전장치·자율 실행 루프를 갖고 시작하도록. 매번 세팅을 다시 만들 필요가 없습니다.

📜 규칙 7🪝 훅 8🤖 에이전트 4 🔁 자율 루프🧪 CI + 테스트🔌 MCP(github·notion)

② 사용법

  1. 생성 — ./new-project.sh myproj (이름에 /·공백·./- 시작은 거부, 한글 허용)
  2. 환경 — 생성된 폴더에서 make setup (venv + 의존성, pyyaml 포함)
  3. 작성 — cp spec.template.yaml spec.yaml 후 프로젝트 설명·결과물·golden 경로 기입
  4. 자율 실행 — Claude Code 에서 /autoloop → 결과물이 golden 에 도달할 때까지 자동 반복
주의 — 같은 이름의 폴더가 이미 있으면 덮어쓰지 않고 중단합니다. gh 미인증이면 GitHub 연동만 건너뛰고 로컬은 정상 생성됩니다.

③ 무엇이 복제되는가

복사는 화이트리스트 방식입니다(필요한 것만 통째로). 런타임 상태(.omc)는 복제 후 자동 제거됩니다.

⑤ Loop Station — 자율 실행이 도는 법

사람이 매 턴 개입하지 않고, 명세(spec) + golden data를 근거로 결과물이 목표에 도달할 때까지 스스로 반복합니다. 판정 오라클은 tools/evaluate.py, 학습 근거는 logs/.

흐름 통과/완료 실패→수정

판정 모드 (게이밍 방지)

golden.mode판정 방식용도
reproduceoutput_path 를 golden.path 와 비교고정 1회성 산출물 재현
generalizeholdout 으로만 최종 판정(holdout_output_path vs holdout_path)변하는 입력을 처리하는 생성기
오라클 게이밍 방지 — generalize 모드에선 holdout golden 의 diff 를 로그에 노출하지 않습니다. 그래서 루프가 "golden 복사"가 아니라 "일반화하는 생성기"로 수렴합니다. threshold < 1.0 이면 모든 모드에서 유사도 기반 통과를 허용합니다.

⑦ 구현 방식 (new-project.sh 내부)

1) 인자 검증

이름에 경로 구분자·공백, ./- 시작을 거부. 기존 폴더가 있으면 덮어쓰지 않고 중단(멱등·안전).

2) 화이트리스트 복사

cp -r 로 폴더 통째 복사 → 새 파일이 자동 포함. 복사 후 .omc 런타임 상태를 prune.

3) 생성물

프로젝트명으로 CLAUDE.md(경로표) · README.md · logs/CHECKPOINT.md 시드 작성.

4) git + 안전망

git init → main/stable 브랜치 → .git/hooks 에 pre-commit 설치(+core.hooksPath=.githooks) → 영어 컨벤션 초기 커밋.

5) GitHub 연동

gh 인증 시 gh repo create --private --push 로 리포 생성 + main/stable 푸시. 미인증이면 건너뜀.

6) 인터프리터 통일

produce·evaluate 모두 venv/bin/python(make 타깃) 사용 — 복제 직후 첫 회차부터 깨지지 않게.

설정 파일 분리 — settings.json=공통 훅(복제됨, Claude Code가 안 건드림) / settings.local.json=권한(복제 안 함, 프로젝트마다 자동 관리). 훅 경로는 $CLAUDE_PROJECT_DIR 로 프로젝트 루트 기준 동작.
