# 프로젝트 구조 (공통)

모든 프로젝트는 아래 표준 폴더 구조를 따른다.

```
프로젝트/
├── CLAUDE.md          # 이정표(경로 지도)만. 규칙·절차를 직접 쓰지 않는다.
├── README.md          # 프로젝트 개요·실행 방법
├── .gitignore
├── .claude/
│   ├── agents/        # 에이전트 역할·입출력·도구 정의 (.md)
│   ├── hooks/         # 훅 실행 스크립트
│   └── settings.local.json  # 훅 등록, 권한 정책
├── rules/             # 코딩 컨벤션, 도메인 규칙
├── docs/              # 아키텍처, 워크플로, 명세
├── logs/              # 에러·이터레이션 이력
└── src/               # 구현 소스 코드
```

| 폴더 | 저장 내용 |
|------|-----------|
| `.claude/agents/` | 에이전트 역할·입출력·도구 정의 |
| `.claude/hooks/` | 훅 실행 스크립트 |
| `rules/` | 코딩 컨벤션, 접근 정책, 도메인 규칙 |
| `docs/` | 아키텍처 설계, 명세, 워크플로 |
| `logs/` | 실패·에러 기록, 이터레이션 이력 |
| `src/` | 구현 코드 |

## 금지 사항
- `src/` 이외 경로에 구현 코드를 두는 것
- CLAUDE.md에 규칙·절차를 직접 작성하는 것 (CLAUDE.md는 이정표만)
- 에이전트 정의를 CLAUDE.md나 `rules/`에 작성하는 것 (반드시 `.claude/agents/`에)
- 에러 발생 시 `logs/`에 기록하지 않고 넘어가는 것

## mcp 참조
- MCP 설정은 각 프로젝트에 복사하지 않는다. 전역 디렉토리의 `mcp/`를 참조한다.
