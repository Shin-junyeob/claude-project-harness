# 컨텍스트 관리 규칙 (공통)

> 목표: **큰 컨텍스트 유지가 아니라** "린 컨텍스트 + 외부 영속 상태 + 잦은 능동 compact + 재읽기".
> (상세 근거 분석은 전역 하네스의 `docs/context_management_plan.md` 참고 — 복제본에는 포함되지 않음)

## 원칙
- **auto-compact를 끄지 않는다**(하드리밋 충돌 방지용 안전망). 1M 윈도우는 헤드룸으로만 두고 **채우는 것을 목표로 삼지 않는다** — in-window가 커질수록 일관성(recall)이 오히려 떨어진다.
- **진실의 출처는 디스크다.** 작업 상태를 대화가 아니라 외부 파일에 박제한다.

## 체크포인트 (핵심)
- **마일스톤마다 기록**한다: 스토리 완료 / 큰 결정 / 실패 분석 시.
- 위치:
  - `logs/CHECKPOINT.md` — 현재 작업 진행 상태(주 기록처).
  - 영속이 꼭 필요한 핵심만 루트 `CLAUDE.md` 또는 auto memory(`MEMORY.md`)로. **compact 후 자동 재로드**되는 곳이기 때문.
  - **의존 금지(compact 시 손실)**: `#` 인-대화 메모, path-scoped rules, 중첩 `CLAUDE.md`(해당 파일을 다시 읽기 전까지).
- 체크포인트 형식:
  ```
  ## <시각/마일스톤>
  - 한 일:
  - 다음 할 일:
  - 핵심 결정·제약:
  - 관련 파일:
  - (실패 시) 가설:
  ```

## 능동 compact
- 큰 컨텍스트가 쌓이기 전, 마일스톤마다 `/compact focus <보존할 핵심>` 으로 요약 범위를 **직접 지정**한다(자동 추측에 맡기지 않는다).
- 임계값을 굳이 조정한다면 `CLAUDE_AUTOCOMPACT_PCT_OVERRIDE` 로 **낮춰** 더 자주 compact 한다(올리지 말 것 — 역효과).

## 재진입 규약
- compact/재시작 직후 **가장 먼저** `logs/CHECKPOINT.md`(+ `spec.yaml`, `logs/iterations.md`, PRD 상태)를 **재읽기**하고 그 지점부터 이어간다.

## 대용량 위임
- 큰 파일·로그·검색은 subagent 에 위임하고, 메인 컨텍스트에는 **결론만** 남긴다.
